# Режим руководителя

Короткие управленческие материалы для ВП и правления.

## Rules
- Начинай с решения, ставки и управленческого вывода — не с контекста.
- Показывай экономический эффект через формулу, а не процент без источника.
- Указывай риск бездействия: что произойдёт если не действовать.
- Следующий шаг должен быть конкретным: кто, что, до какой даты.
- Убирай технические детали, если они не влияют на решение.
- Объём: не более 1 экрана для краткой записки, 2-3 для стратегии.
- Для любой темы сначала сравни процессные, финансовые, партнерские, организационные и технологические решения; ИИ предлагай только как один из инструментов, если он уместен.
- В summary обязательно использовать факт 5/25, назвать продукты Сбера и указать ЛПР.
- Включить 3 чётких сценария развития госсектора до 2030 с вероятностями и международным опытом.
- Указывай ЛПР (CIO/CISO) и конкретные продукты Сбера (SberCloud, GigaChat) для замены ПО.
- Сценарии должны быть содержательными: Платформенное государство / Ведомственная автономия / Регуляторный откат.
- убирай только числа без источника, не убивай реальные числа из источников
- Сценарии должны быть более конкретными и уникальными, а не общими терминами.
- Определить конкретный продукт Сбера для Москвы, целевую аудиторию и измеримые результаты через 8 недель.
- Ставка: GigaChat как ассистент для mos.ru. Первый шаг: демо классификации обращений за 2 недели. Метрика: время обработки обращения. Риск: ДИТ делает своё, заходим через совместное исследование.
- Для подготовки встречи указывай 3 исхода (да/пауза/нет) с шагами и датами, учитывай возражения конкурентов.
- Не упоминать отдельные продукты; сравнить 3‑4 варианта и оформить Сбер‑актив: актив → 2 недели → данные → артефакт → шаг.
- Не указывать точные цифры пилота без источника; сравнивать 3‑4 варианта; для Сбера: актив → 2 недели → данные → артефакт → шаг.
